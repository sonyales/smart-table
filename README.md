# Smart Table

**Smart Table** — интерактивное веб-приложение для работы с табличными данными о продажах.

Проект позволяет просматривать набор записей, выполнять поиск, применять фильтры, сортировать данные по ключевым полям и управлять количеством строк на странице. Интерфейс обновляется динамически: пользователь меняет параметры таблицы, а приложение формирует актуальный запрос, получает данные и перерисовывает результат без перезагрузки страницы.

## О проекте

Smart Table решает типичную задачу административных интерфейсов: удобная навигация по большому объёму данных.

В приложении реализована таблица продаж с данными о дате операции, покупателе, продавце и сумме заказа. Пользователь может быстро найти нужные записи, отфильтровать таблицу по продавцу, отсортировать данные и перейти к нужной странице результатов.

Проект написан на JavaScript без UI-фреймворков. Логика разделена на независимые модули, каждый из которых отвечает за отдельную часть интерфейса: таблицу, поиск, фильтрацию, сортировку, пагинацию и работу с данными.

## Функциональность

- Загрузка данных с удалённого API
- Отображение записей о продажах в таблице
- Поиск по данным таблицы
- Фильтрация записей по продавцу
- Сортировка по дате
- Сортировка по сумме заказа
- Переключение направления сортировки
- Сброс неактивных сортировок при выборе новой колонки
- Пагинация результатов
- Переход на первую, предыдущую, следующую и последнюю страницу
- Выбор количества строк на странице
- Отображение диапазона текущих записей
- Отображение общего количества найденных записей
- Сброс применённых фильтров
- Динамическая перерисовка таблицы при изменении параметров

## Стек технологий

- **JavaScript ES Modules** — модульная организация кода
- **Vite** — быстрый dev-сервер и production-сборка
- **HTML5** — структура страницы и шаблоны интерфейса
- **CSS3** — стилизация таблицы и элементов управления
- **Fetch API** — получение данных с сервера
- **URLSearchParams** — формирование query-параметров для API-запросов
- **DOM API** — динамическая отрисовка интерфейса без UI-фреймворков
- **GitHub Actions** — автоматическая проверка проекта при изменениях в репозитории

## Архитектура

Приложение построено на модульной архитектуре. Каждый модуль отвечает за отдельную задачу и предоставляет наружу только необходимые методы.

Основной поток работы приложения:

1. Пользователь изменяет состояние интерфейса: вводит поисковый запрос, выбирает фильтр, нажимает на сортировку или меняет страницу.
2. Приложение собирает текущее состояние формы.
3. Модули поиска, фильтрации, сортировки и пагинации преобразуют состояние в параметры запроса.
4. Модуль данных отправляет запрос к API и получает актуальный список записей.
5. Таблица перерисовывается на основе новых данных.
6. Пагинация обновляет диапазон отображаемых записей и общее количество результатов.

Такой подход делает приложение расширяемым: отдельные функции таблицы можно дорабатывать независимо друг от друга.

## Основные модули

### `main.js`

Точка входа приложения.

Файл инициализирует API, таблицу, пагинацию, сортировку, фильтрацию и поиск. Также здесь находится функция `render`, которая собирает текущее состояние интерфейса, формирует параметры запроса и обновляет таблицу.

### `data.js`

Модуль отвечает за работу с данными:

- получает справочники продавцов и покупателей;
- запрашивает записи о продажах;
- преобразует данные API в формат, удобный для отображения в таблице;
- кеширует последний результат, чтобы не выполнять повторный запрос при неизменных параметрах.

### `components/table.js`

Модуль таблицы отвечает за:

- клонирование HTML-шаблонов;
- подключение блоков управления до и после таблицы;
- обработку событий формы;
- создание строк таблицы;
- обновление содержимого таблицы при получении новых данных.

### `components/searching.js`

Модуль поиска добавляет поисковый параметр в запрос, если пользователь ввёл значение в поле поиска.

### `components/filtering.js`

Модуль фильтрации отвечает за:

- заполнение выпадающих списков значениями из справочников;
- обработку очистки отдельных полей;
- формирование параметров фильтрации для API-запроса.

### `components/sorting.js`

Модуль сортировки управляет состоянием сортируемых колонок.

Он переключает направление сортировки, сбрасывает сортировку у остальных колонок и добавляет в запрос параметр вида `field:direction`.

### `components/pagination.js`

Модуль пагинации отвечает за:

- расчёт количества страниц;
- отображение видимых номеров страниц;
- переход между страницами;
- обновление диапазона отображаемых строк;
- передачу в запрос параметров `page` и `limit`.

### `lib/utils.js`

Файл содержит переиспользуемые вспомогательные функции:

- клонирование HTML-шаблонов;
- сбор именованных DOM-элементов;
- преобразование `FormData` в обычный объект;
- создание индексов по коллекциям;
- расчёт страниц для пагинации.

### `lib/sort.js`

Файл содержит функции-компараторы и карту переключения состояний сортировки.

## Работа с данными

Приложение получает данные с удалённого API.

Основные типы данных:

- **records** — записи о продажах;
- **sellers** — справочник продавцов;
- **customers** — справочник покупателей.

Перед отображением записи преобразуются в формат таблицы:

- `id` — идентификатор записи;
- `date` — дата продажи;
- `seller` — имя продавца;
- `customer` — имя покупателя;
- `total` — сумма продажи.

## Пользовательский сценарий

1. При открытии приложения загружаются справочники продавцов и покупателей.
2. Таблица получает первую страницу записей.
3. Пользователь меняет параметры отображения: поиск, фильтр, сортировку или страницу.
4. Приложение формирует новый набор query-параметров.
5. Данные запрашиваются с API.
6. Таблица и пагинация обновляются в соответствии с полученным результатом.

## Установка и запуск

Склонируйте репозиторий:

```bash
git clone https://github.com/sonyales/smart-table.git
cd smart-table
```

Установите зависимости:

```bash
npm install
```

Запустите проект в режиме разработки:

```bash
npm run dev
```

После запуска приложение будет доступно по локальному адресу, который Vite выведет в терминале.

## Сборка проекта

Для production-сборки выполните:

```bash
npm run build
```

Собранные файлы будут помещены в директорию `dist`.

## Предпросмотр production-сборки

```bash
npm run preview
```

## Особенности реализации

- Приложение написано на нативном JavaScript без UI-фреймворков
- Интерфейс строится на HTML-шаблонах и DOM API
- Логика таблицы разделена на независимые модули
- Параметры поиска, фильтрации, сортировки и пагинации собираются в единый query-объект
- Данные запрашиваются с API только после изменения состояния интерфейса
- Последний результат запроса кешируется
- Справочники продавцов и покупателей загружаются отдельно и используются для подготовки данных
- Таблица поддерживает динамическую перерисовку строк
- Пагинация показывает диапазон отображаемых записей и общее количество результатов
